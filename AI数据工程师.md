# 具身智能AI数据工程师

## 一、核心定义

**具身智能AI数据工程师**是站在**机器人学、人工智能与数据工程交叉点**上的专业角色，负责**将物理世界的交互经验转化为可被模型高效学习的结构化数据资产**。其本质是为"有身体的AI"（如人形机器人、机械臂、无人车）提供高质量训练数据的"燃料供给者"。

> **角色定位**：连接物理世界与机器学习系统的数据架构师。与传统AI数据工程不同，该岗位直接服务于**Perception–Planning–Control**闭环，对具身智能模型的**安全性、泛化性和可落地性**具有决定性影响。

## 二、核心数据类型

具身智能的数据不是孤立"样本"，而是**时空连续的物理经验**，具有**多模态、强时序、物理约束**三大特征：

| 数据类别 | 具体内容 | 关键特征 |
|----------|----------|----------|
| **感知数据 (Perception)** | RGB/RGB-D图像、LiDAR点云、IMU、触觉/力传感器、环境声 | 高频率、强噪声、数据量巨大 |
| **动作与控制数据 (Action/Control)** | 关节角/速度/力矩、末端执行器轨迹、控制指令序列 | 与硬件强耦合，受物理约束 |
| **环境与状态数据 (State/World)** | 物体6-DoF位姿、坐标系变换、碰撞/接触信息、场景结构 | 需高精度空间校准 |
| **任务与语义数据 (Task/Semantic)** | 自然语言指令、任务分解图、成功/失败标签、RL奖励信号 | 连接人类意图与机器行为 |
| **仿真与合成数据 (Synthetic)** | 虚拟环境生成的带完美标注数据、域随机化场景、Corner Case | 低成本扩展物理经验 |

## 三、核心工作内容

### 1. **数据采集系统与工具链设计**
- 构建真实机器人(ROS/ROS2)与仿真环境(Isaac Sim/MuJoCo)的数据采集架构
- 设计传感器网络的时间同步与空间标定方案
- 开发自动化任务生成与技能-物品-场景体系

### 2. **多模态数据清洗、对齐与校正**
- 跨模态时序对齐(图像-点云-动作)
- 噪声过滤、漂移校正与异常检测
- 保障数据物理合理性(防穿模、重力约束)

### 3. **数据切分与结构化(核心能力)**
- 将连续交互流切分为**Episode/Trajectory/Skill**单元
- 基于接触点、状态变化、任务边界进行语义切分
- 适配不同训练范式(模仿学习/强化学习/VLA模型)的数据组织结构

### 4. **数据标注与质量评估体系**
- 管理3D标注、轨迹标注、动作序列等复杂标注任务
- 构建自动化标注流水线(结合SAM/GPT-4V等大模型)
- 评估数据覆盖度、多样性、分布偏置

### 5. **数据基础设施与版本管理**
- 设计PB级高吞吐时序数据存储架构(对象存储+向量数据库)
- 实现数据血缘追踪与版本控制系统
- 优化训练数据加载Pipeline(Parquet/Zarr/HDF5)

### 6. **数据闭环与算法协同**
- 根据模型失败案例反向设计数据采集策略
- 优化仿真-真实数据配比(Sim-to-Real)
- 建立"数据版本-模型性能"关联分析体系

## 四、技能要求

### 1. **基础技术栈(必备)**
- **编程语言**：精通Python(数据处理核心)，掌握C++(性能关键模块)
- **机器人中间件**：ROS/ROS2(消息机制、Bag包处理)
- **数据处理**：Pandas/NumPy，OpenCV/Open3D，Spark/Ray
- **存储技术**：Parquet/HDF5/Zarr格式，MinIO/S3，向量数据库(Milvus)

### 2. **机器人与仿真能力(关键区分项)**
- 机器人传感器与执行器原理
- 坐标系、运动学、物理约束基础
- 仿真平台：NVIDIA Isaac Sim(主流)，MuJoCo/Gazebo/PyBullet
- "**必须听得懂机器人在说什么**"

### 3. **AI与数据工程能力**
- 熟悉PyTorch/TensorFlow数据接口
- 理解VLA(视觉-语言-动作)模型、多模态融合、世界模型的数据需求
- 熟悉强化学习/模仿学习的数据格式要求(如D4RL)
- 高吞吐数据Pipeline设计与分布式处理

### 4. **工程与软技能**
- Linux/Shell/Docker/Kubernetes
- 数据版本控制与实验管理
- 跨团队协作(与算法、机器人硬件团队)
- 物理直觉与对数据真实性的敏锐洞察
